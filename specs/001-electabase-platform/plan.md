# Implementation Plan: ElectaBase — Civic-Tech Political Transparency Platform

**Branch**: `001-electabase-platform` | **Date**: 2026-05-08 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `specs/001-electabase-platform/spec.md`

## Summary

ElectaBase is a full-stack Next.js web application serving as India's authoritative political
transparency database. The v1 scope covers: (a) ALL Rajya Sabha politicians — all states,
all years; (b) ALL Lok Sabha politicians from Tamil Nadu — all Tamil Nadu constituencies, all
years. Lok Sabha coverage for other states is explicitly deferred to future versions, with the
architecture designed for zero-re-architecture expansion via scraper scope configuration.
The system is built around an automated scraping pipeline ingesting from five government and
civic portals (ECI, Lok Sabha, Rajya Sabha, PRS, ADR) on a weekly schedule, with every data
point traceable to a clickable authentic source. Two deterministic scoring algorithms
(Effectiveness and Corruption) provide comparable, auditable rankings.

## Technical Context

**Language/Version**: TypeScript 5.x (strict mode)
**Primary Dependencies**:
- Next.js 15 (App Router) — SSR + SSG + API routes
- Prisma 5 — ORM, schema-first, PostgreSQL
- Supabase — PostgreSQL database + Auth (magic link) + Storage (photos)
- Playwright — web scraping (JS-rendered government portals)
- Cheerio — HTML parsing (static sources: PRS, ADR)
- pdf-parse — PDF extraction (ADR affidavits)
- Resend + React Email — transactional notifications
- `@upstash/ratelimit` + Upstash Redis — correction form rate limiting
- Cloudflare — CDN, HTTPS, DDoS protection (free tier)

**Storage**: PostgreSQL (Supabase, primary data) + Supabase Storage (politician photos, 1 GB free)
**Testing**: Vitest (unit), Playwright (e2e scraper validation)
**Target Platform**: Vercel (Next.js hosting, free tier) + Supabase (free tier)
**Project Type**: Full-stack web application (Next.js App Router, monorepo)
**Performance Goals**: LCP < 2.5s, FID < 100ms, CLS < 0.1; search p95 < 2s; compare < 3s; PageSpeed mobile ≥ 90
**Constraints**: All tooling free-tier at MVP; WCAG 2.1 AA; DPDP Act 2023 (India) compliance for personal data
**Scale/Scope**: ~3,000–3,500 politicians in v1 (Rajya Sabha all states ~2,500 + Lok Sabha Tamil Nadu ~500–700); 100K+ concurrent users at scale; 5 scraping sources (Lok Sabha source scoped to Tamil Nadu via config); 44 FRs across 13 feature groups; coverage expandable to new Lok Sabha states via DB config change only

**Structure Decision**: Web application monorepo using Next.js App Router. Public pages and
admin panel coexist in a single Next.js project. API routes are colocated in `app/api/`.
Business logic (scrapers, scoring, pipeline) lives in `lib/` for clean separation from
presentation. No separate backend service needed at MVP — the Next.js API routes + GitHub
Actions cron replace a dedicated worker.

## Phase 0: Research Summary

All technical decisions are documented in [research.md](./research.md). Key decisions:

| Decision | Choice | Rationale |
|----------|--------|-----------|
| Web scraping | Playwright + Cheerio | Playwright for JS-rendered portals; Cheerio for static HTML |
| PDF parsing | pdf-parse | Standard for Indian election affidavit PDFs (ADR/MyNeta) |
| Scheduler | GitHub Actions cron | Free tier, no extra infrastructure; calls `/api/admin/refresh` |
| Auth | Supabase Auth (magic link) | Colocated with DB; no custom auth; admin-only, no self-registration |
| Search | PostgreSQL FTS + GIN index | Sufficient for ~15K politicians; upgrade path to Meilisearch at scale |
| Email | Resend | 3,000/month free; TypeScript SDK; React Email templates |
| Storage | Supabase Storage | Colocated with DB; 1 GB free; sufficient for politician photos at MVP |
| Rate limiting | Upstash Redis | Free tier; works across multiple Vercel instances |
| Ad zones | PostgreSQL `AdZone` table | Simple config, cached; no Vercel-specific dependency |
| Effectiveness score | Log-normalized formula, `effectiveness-v1.0` | 7 components, max 100 pts, gaming-resistant |
| Corruption score | Severity-tiered formula, `corruption-v1.0` | 6 components, IPC severity tiers, acquittal reduces score |
| Slug format | `{name}-{birth-year}[-{n}]` | Stable, human-readable, collision-safe |
| v1 scraping scope | Rajya Sabha: all states; Lok Sabha: Tamil Nadu only | Controlled via `DataSource.scraperConfig.scope.states`; new states added by config change only |
